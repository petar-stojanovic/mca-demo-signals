NgRx SignalStore

One file. One API. Everything in the store.

Why SignalStore?

Classic NgRx is powerful but requires a lot of ceremony. SignalStore replaces all of that with a single, functional API built on Angular Signals.

Classic NgRx — 4 files
  • actions.ts
  • reducer.ts
  • selectors.ts
  • effects.ts
SignalStore — 1 file
  • withState — define state
  • withComputed — derive values
  • withMethods — update state
  • withHooks — lifecycle

4 rxMethod

A built-in SignalStore tool — not something you write. Connects RxJS streams to your store.

Accepts a static value, a signal, or an observable. No manual .subscribe(). Cleans up automatically when the store is destroyed.

No .subscribe() Reactive by default Auto-cleanup on destroy
You will see rxMethod in action on the Custom Features page.
