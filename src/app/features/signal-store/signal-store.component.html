NgRx SignalStore

One file replaces four. State management built on Angular Signals.

Why SignalStore?

Classic NgRx is a proven pattern for large-scale Angular apps. But it has a cost: every feature requires four files working in concert. SignalStore replaces all of that with a single, functional API built on Angular Signals.

Classic NgRx — 4 files
actions.ts 
// Name every user intent as a typed action object
import { createAction, props } from '@ngrx/store';

export const toggleInCart = createAction(
  '[Cart] Toggle Item',
  props<{ itemId: number }>()
);
reducer.ts 
// Define state shape + how each action changes it
import { createReducer, on } from '@ngrx/store';
import { toggleInCart } from './cart.actions';

export interface CartState { cartItems: CartItem[] }

export const initialState: CartState = {
  cartItems: [{ id: 1, name: 'Wireless Headphones', ... }],
};

export const cartReducer = createReducer(
  initialState,
  on(toggleInCart, (state, { itemId }) => ({
    ...state,
    cartItems: state.cartItems.map(item =>
      item.id === itemId
        ? { ...item, inCart: !item.inCart }
        : item
    ),
  }))
);
selectors.ts 
// Derive values from state — memoized, composable
import { createFeatureSelector, createSelector } from '@ngrx/store';

const selectCart = createFeatureSelector<CartState>('cart');
const selectItems = createSelector(selectCart, s => s.cartItems);

export const selectCartOnlyItems = createSelector(
  selectItems, items => items.filter(i => i.inCart)
);
export const selectCartTotal = createSelector(
  selectCartOnlyItems,
  items => items.reduce((sum, i) => sum + i.price, 0)
);
effects.ts 
// Async operations and side effects — separate class, separate file
import { inject, Injectable } from '@angular/core';
import { Actions, createEffect, ofType } from '@ngrx/effects';

@Injectable()
export class CartEffects {
  private actions$ = inject(Actions);
  private http = inject(HttpClient);

  loadProducts$ = createEffect(() =>
    this.actions$.pipe(
      ofType(loadProducts),
      switchMap(() =>
        this.http.get<CartItem[]>('/api/products').pipe(
          map(products => loadProductsSuccess({ products }))
        )
      )
    )
  );
}
SignalStore — 1 file
cart.store.ts 
export const CartStore = signalStore(
  { providedIn: 'root' },

  // replaces: initialState in reducer.ts
  withState({
    cartItems: [{ id: 1, name: 'Wireless Headphones', ... }],
  }),

  // replaces: selectors.ts
  withComputed(({ cartItems }) => ({
    cartOnlyItems: computed(() => cartItems().filter(i => i.inCart)),
    cartTotal: computed(() =>
      cartItems().filter(i => i.inCart)
        .reduce((sum, i) => sum + i.price, 0)
    ),
  })),

  // replaces: actions.ts + reducer.ts on() handler
  withMethods((store) => ({
    toggleInCart(itemId: number) {
      patchState(store, state => ({
        cartItems: state.cartItems.map(item =>
          item.id === itemId
            ? { ...item, inCart: !item.inCart } : item
        ),
      }));
    },
  })),

  // replaces: effects.ts lifecycle + withEventHandlers for side effects
  withHooks({
    onInit: (store) => console.log('Store ready', store),
    onDestroy: (store) => console.log('Cleanup', store),
  }),
);
Classic NgRx file Purpose SignalStore equivalent
actions.ts Name every user intent withMethods
reducer.ts Define state + apply changes withState + withMethods
selectors.ts Derive computed values withComputed
effects.ts Async ops + side effects rxMethod or effect

1 withState

Define what your app needs to remember. Every property becomes a live signal automatically.

You describe the shape of your data once. SignalStore turns each property into a signal — a reactive value that updates the UI whenever it changes. No boilerplate, no decorators.

NgRx — reducer.ts initial state 
// reducer.ts — state shape lived here, mixed with mutation logic
export interface CartState { cartItems: CartItem[] }
export const initialState: CartState = { cartItems: [] };
// To read in a component: inject Store, then store.select(selectCartItems)
SignalStore — withState 
export const CartStore = signalStore(
  withState({
    cartItems: [
      { id: 1, name: 'Wireless Headphones', price: 59.99, inCart: false },
      { id: 2, name: 'Mechanical Keyboard',  price: 89.99, inCart: false },
      // ...
    ],
  }),
  // cartItems is now a signal: store.cartItems()
);

2 withComputed

Derived values that stay in sync — no manual recalculation.

Computed values are calculated from existing state. When the source changes, computed values update automatically. Add items to the cart or search in Section 3 to see them update here.

NgRx — selectors.ts 
// selectors.ts — composable, memoized, but verbose
const selectCart = createFeatureSelector<CartState>('cart');
const selectItems = createSelector(selectCart, s => s.cartItems);
export const selectCartTotal = createSelector(
  selectItems,
  items => items.filter(i => i.inCart).reduce((sum, i) => sum + i.price, 0)
);
SignalStore — withComputed 
withComputed(({ cartItems, searchQuery }) => ({
  cartOnlyItems: computed(() => cartItems().filter(item => item.inCart)),
  cartTotal: computed(() =>
    cartItems().filter(item => item.inCart)
      .reduce((sum, item) => sum + item.price, 0)
  ),
  // reacts to TWO signals at once — searchQuery AND cartItems
  filteredItems: computed(() =>
    cartItems().filter(item =>
      item.name.toLowerCase().includes(searchQuery().toLowerCase())
    )
  ),
}))

3 withMethods

All state changes go through one place. Components just call methods.

Methods are the only way to update state. Try searching and toggling items — watch Sections 1 and 2 update reactively.

NgRx - actions.ts + reducer.ts 
// Component dispatches an action...
this.store.dispatch(toggleInCart({ itemId }));

// ...NgRx routes it to the matching on() handler in reducer.ts

  ...state,
  cartItems: state.cartItems.map(item =>
    item.id === itemId ? { ...item, inCart: !item.inCart } : item

4 rxMethod

A bridge between RxJS streams and your store — accepts a value, a signal, or an observable, and cleans up automatically.

When you need to handle async operations like HTTP calls, rxMethod connects an RxJS pipeline to your store without manual subscription management.

No .subscribe() Reactive by default Auto-cleanup on destroy
NgRx - effects.ts 
// effects.ts — an entirely separate class just for async operations
@Injectable()
export class CartEffects {
  private actions$ = inject(Actions);
  private http = inject(HttpClient);

  loadProducts$ = createEffect(() =>
    this.actions$.pipe(
      ofType(loadProducts),
      switchMap(() => this.http.get('/api/products'))
    )
  );
}
SignalStore - rxMethod
