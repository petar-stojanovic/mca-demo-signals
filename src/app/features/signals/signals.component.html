How Angular Detects Changes

Zone.js told Angular when to check for changes. Signals tell Angular what changed — no more checking the whole tree.

1

Default CD

Any event (click, timeout, HTTP) triggers a full tree traversal. Every component is checked top to bottom — even those unrelated to what changed.

Checks all N components
2

OnPush

Angular skips entire subtrees unless a component is marked dirty. Still traverses the path from root and re-evaluates all expressions in the dirty component.

Checks the dirty path
3

Signals

Angular tracks exactly which template binding reads which signal. When a signal changes, only those DOM nodes update — no traversal, no dirty marking, no guessing.

Updates 1 binding
